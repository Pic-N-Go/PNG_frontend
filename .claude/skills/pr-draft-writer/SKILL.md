# PR Draft Writer Skill

팀 템플릿에 맞는 PR 제목/본문 초안을 생성하고, 사용자 확인을 받은 뒤 현재 작업 브랜치에 실제 PR을 생성하는 스킬입니다.

## 목적

- `.github/pull_request_template.md` 형식에 맞는 PR 초안을 일관되게 생성한다.
- 리뷰어가 바로 검토 가능한 테스트 절차/체크리스트를 포함한다.
- 초안 내용을 사용자에게 보여주고 **승인받은 경우에만** 실제 PR을 생성한다.

## 입력

- 현재 변경사항 (`git status`, `git diff HEAD`, `git log`)
- 현재 브랜치와 원격 추적 브랜치 여부 (`git branch --show-current`, `git rev-parse --abbrev-ref --symbolic-full-name @{u}`)
- 필요 시 사용자 전달 컨텍스트 (`$ARGUMENTS`)

## 동작 규칙

1. `.github/pull_request_template.md`를 읽고 섹션 구조를 그대로 유지한다.
2. 변경사항 기반으로 아래를 채운다.
   - 개요
   - 관련 이슈 (없으면 `없음` 또는 TODO)
   - 작업 내용 체크박스
   - 테스트 방법 (재현 가능한 단계)
   - 스크린샷 (UI 변경 없으면 섹션 삭제, 있으면 `TODO:` 로 마킹)
   - 체크리스트 (PR 템플릿 항목 그대로 유지)
3. 해당 없는 섹션은 빈 칸으로 남기지 않고 섹션째로 삭제한다. 불확실한 정보(예: 테스트 결과, CI 통과 여부)는 `TODO:` 로 명시한다.
4. 초안(제목 + 본문)을 사용자에게 그대로 보여주고 승인 여부를 확인한다. 이 단계 전까지는 PR을 생성하거나 push하지 않는다.
5. 승인 시:
   - 현재 브랜치가 원격에 없거나 로컬 커밋이 원격보다 앞서 있으면 `git push` (신규 브랜치는 `-u origin <branch>`) 로 반영한다.
   - `gh pr create --title "<제목>" --body "<승인된 본문>" --base <기본 브랜치>` 로 PR을 생성한다. 기본 브랜치는 별다른 지시가 없으면 저장소의 default branch(`main`)로 한다.
   - 생성된 PR URL을 보고한다.
6. 승인 거부 또는 수정 요청 시:
   - 요청받은 대로 초안을 수정해 다시 확인받는다.
   - 수정 없이 보류를 원하면 `docs/ai/pr-drafts/<branch>-pr.md` 로 파일만 저장하고 종료한다 (실제 PR 생성 없음). 파일로 남기는 경우, `git ls-files --others --exclude-standard` 로 파일이 최종적으로 남는지 사용자에게 알린다.
7. 승인 후 실제 PR 생성에 성공했다면, 4-5단계에서 임시로 만든 초안 파일(있는 경우)은 남겨두지 않고 삭제한다 — PR 자체가 최신 기록이므로 중복 파일을 남기지 않는다.

## 출력 형식

1) Draft Title
2) Draft Body (승인 요청 시점의 전체 본문)
3) 승인 여부 확인 질문
4) (승인 후) 생성된 PR URL 또는 (거부 시) 저장된 초안 파일 경로
