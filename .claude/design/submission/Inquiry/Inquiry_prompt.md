# 1:1 문의 화면 작업 요청

## 진입 경로
- **FAQ 화면 하단 `[문의하기]` CTA** → `InquiryCompose` (작성 폼)
- **마이페이지 > 1:1 문의** → `InquiryList` (내 문의 내역)
- **알림 탭 (답변 도착)** → `InquiryDetail` (상세)
- 3개 스크린을 하나의 `InquiryStack`으로 구성. 헤더는 마이페이지와 동일한 뒤로가기 + 중앙 타이틀.

## 스택 흐름
```
FAQ → [문의하기] → InquiryCompose ──제출──▶ InquiryList (justSubmitted 플래그)
MyPage → 1:1 문의 ─────────────────────▶ InquiryList
InquiryList → 항목 tap ─────────────────▶ InquiryDetail
알림(답변 도착) ─────────────────────▶ InquiryDetail (deep link)
```

## 주요 스펙

### 공통
- 뷰포트: 390 × 844 기준, 콘텐츠 좌우 패딩 **28px**
- 컬러: 마이페이지 팔레트 그대로 (`#fff` / `#f5f5f7` / `#e31b59`)
- 주 버튼(제출/새 문의): pill 999, height **52**, `#e31b59` / #fff / 15·600
- 상태 배지: pill 999, height 22, 11·600
  - 답변 대기: bg `#fff2d9` / fg `#b06f00`
  - 답변 완료: bg `#e0f0dc` / fg `#5a9855`

### 2a · InquiryCompose (문의 작성)
- 상단 안내 배너: `#fff5f8` bg + `IconInfoCircle(#e31b59)` — "영업일 기준 1~2일 이내 답변" 명시
- 필드 (라벨 13·600 + `*` 브랜드색 필수 표시):
  - **문의 유형** (필수) — 드롭다운. 인라인 확장으로 옵션 노출 (별도 시트 X, 짧은 목록이라)
  - **제목** (필수, max 50자)
  - **내용** (필수, multiline, min-height 140, max 1000자, 우측 상단 카운터)
  - **사진 첨부** (선택, 최대 3장) — 64×64 dashed placeholder + 썸네일. 우상단 X로 개별 삭제
  - **답변 받을 이메일** — 로그인 계정 자동 채움, 편집 가능, `keyboardType="email-address"`
- 유형 옵션: `촬영 스팟 관련 / 사진·업로드 / 앱 오류/버그 / 계정·보안 / 이용안내 / 기타`
- 하단 고정 `[보내기]` — 필수 3개 미충족 시 `rgba(0,0,0,0.12)` disabled
- 제출 성공 시 `navigation.replace('InquiryList', { justSubmitted: true })` (뒤로가기로 폼 재진입 방지)

### 2b · InquiryList (내 문의 내역)
- 상단 필터 칩(가로 스크롤, 스크롤바 숨김): `전체 N / 답변 대기 N / 답변 완료 N`
  - 활성: `#e31b59`/#fff · 비활성: `#fff` + 1px `rgba(0,0,0,0.08)`
- 카드: `#f5f5f7`, 라디우스 16, 패딩 16/18, 리스트 gap 10
  - 상단: 상태 배지 + (미확인 답변이면 `#e31b59` 도트 + "NEW" 라벨) · 우측 날짜 11 / text2
  - 중단: 제목 14 (대기·미확인 600, 확인함 500 · text 72%)
  - 하단: 카테고리 12 / text2
- 하단 고정 `[+ 새 문의 작성]` pill CTA
- 빈 상태: "문의 내역이 없어요 / 궁금한 점이 있다면 새 문의를 남겨보세요."

### 2c · InquiryDetail (상세 & 답변)
- 헤더 영역: 상태 배지 + 날짜·카테고리 → 제목 18 / 700
- **내 문의 카드** (`#f5f5f7`): 아바타(Y, 검정) + "나" · 시간 → 본문 14 / text 78% → 첨부 썸네일(56)
- **관리자 답변 섹션**:
  - 라벨: 2×12 세로 바 + "관리자 답변" 12·600 브랜드색
  - 카드: `#fff5f8` + 1px `#fbd6e2` 보더 · 아바타(P, 브랜드) + "PNG 운영팀"
- 답변 카드 아래 **피드백 바**: `#f5f5f7` 컨테이너, "답변이 도움이 됐나요?" + 👍 도움됨 pill + 👎 pill
  - 선택 시 브랜드 필 상태 (토글, 재클릭 해제)
- **[답글 보내기]** 아웃라인 pill — `InquiryReply` 화면(스크린 별도, 이번 라운드 미포함)
- 답변 없을 때(`inquiry.answer == null`): 관리자 답변 섹션 전체 숨김, 카드 하단에 "답변 대기중입니다" 미니 배너만 노출

## 상태 규칙
| 상태 | 조건 | 리스트 표기 | 상세 표기 |
|---|---|---|---|
| `waiting` | 관리자 답변 X | 옐로 배지 · 제목 600 | 답변 섹션 없음 |
| `answered · unread` | 답변 O, 사용자 미열람 | 그린 배지 + NEW 도트 · 제목 600 | 진입 시 서버에 read 처리 |
| `answered · read` | 답변 O, 열람함 | 그린 배지 · 제목 500 / 72% | 답변 카드 노출 |

- 관리자가 답변을 등록하면 서버 푸시(FCM/APNs)로 알림 → 알림 tap 시 `InquiryDetail` 딥링크
- 답글 보내기는 다중 라운드 스레드 (추후 라운드에서 구현)

## 유의사항
- 인라인 `style` + `normalize()` 유지 (StyleSheet.create 금지)
- 아이콘: `@tabler/icons-react-native` · strokeWidth 2 · 크기 14/16/18/20/24
- 카드 그림자/보더 사용 금지 — 배경색 계층으로만 위계
- `KeyboardAvoidingView` 사용 (compose 화면)
- 사진 첨부는 `react-native-image-picker` 연동 예정 (현재는 mock URI)
- 제출 API 미구현 상태 → `TODO: POST /inquiries` 주석 남겨둠
- 리스트 미확인 뱃지는 클라이언트 lastReadAt vs answer.answeredAt 비교로 계산
- 마이페이지 재진입 시 `InquiryList`가 스택 루트 — 뒤로가기 시 마이페이지로

## 시안 (Inquiry.dc.html)
- **2a — 문의 작성 폼**: 유형/제목/내용/첨부/이메일. 상단 SLA 안내 배너.
- **2b — 내 문의 내역**: 필터 칩 + 카드 리스트 + 새 문의 CTA.
- **2c — 문의 상세**: 내 문의(회색) / 관리자 답변(핑크) 위계 + 피드백 + 답글.
