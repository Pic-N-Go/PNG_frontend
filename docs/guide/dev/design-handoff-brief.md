# 디자인 핸드오프 브리프 (design Claude / 디자이너용)

> PNG(Pick N Go) 프론트엔드에 **그대로 떨어지는** 디자인 산출물을 받기 위한 제약 요약.
> 새 화면/컴포넌트 디자인을 요청할 때 이 문서를 함께 전달하세요.

## 0. 프로젝트 한 줄

React Native(Expo, bare) 앱. **모바일 전용**(iOS/Android), 태블릿 미지원. **라이트 모드만.** Apple Design System 톤. 기준 해상도 **390×844(iPhone 15 Pro)**, 지원 폭 360~430dp.

## 1. 디자인 토큰

- **브랜드/액센트**: `#E31B59` (핑크) — CTA·활성 상태·포커스 등 인터랙션 요소 전부. 주 팔레트: Black · White · Pink
  - 예외: **알림 화면(`notification.html`) 필터 탭 활성 상태는 Black(`#000`)** 사용 (의도된 디자인, CLAUDE.md 규칙)
- **배경**: 페이지 `#ffffff`, 카드/인풋 `#f5f5f7`
- **카드(기본 규칙)**: 테두리·그림자 없음, 배경 대비로만 elevation
  - 예외: 최근 편의정보 카드처럼 흰 카드+옅은 테두리(`rgba(0,0,0,0.07)`)+미세 그림자를 쓴 곳도 있음. **테두리/그림자 쓸 거면 명시**해주세요(기본은 무테·무그림자)
- **버튼**: pill(높이의 50% radius), primary 높이 **52px**
- **타이포**: Pretendard Variable, 모든 크기 음수 자간(letter-spacing), **최대 weight 600**(그 이상 금지)
- **레이아웃 여백**: 콘텐츠 좌우 **28px**, 카드 그리드 **20px**
- **정렬**: 기본 좌측 정렬, 히어로/로고 영역만 가운데
- **이모지 UI에서 전면 금지**
- **히어로 그라데이션(골든아워)**: `#1a1530 → #2d1b4e → #8b4a6b → #d4856a → #e8a87c → #f0c89a` (히어로에서만)

## 2. 엔지니어링 제약 (⚠️ 여기서 마찰이 자주 남)

- **스타일**: NativeWind `className`(Tailwind v3) 또는 **인라인 `style` 객체**. `StyleSheet.create()` **금지**
- **함수형 `style` 금지**: `style={({pressed}) => ...}` 형태는 이 프로젝트(NativeWind)에서 적용 안 됨 → **객체 style**로 줄 것
- **폰트 크기·높이에 raw 픽셀 금지**: `src/constants/layout.ts` 상수(`FONT_2XS`~`FONT_2XL`, `BUTTON_HEIGHT` 등) 사용. 스케일 필요한 고정값은 `normalize(n)` (`src/utils/normalize.ts`)
  - **폰트 크기는 8개 토큰만**: `10 · 11 · 13 · 14 · 15 · 17 · 22 · 28px` → `FONT_2XS / FONT_XS / FONT_SM / FONT_MD / FONT_LG / FONT_XL / FONT_2XL`. 그 사이값(`9 · 12 · 16 · 18 · 20px` 등)은 **금지**(`12`는 `11` 또는 `13`으로). `14px`만 상수가 없어 `normalizeFontSize(14)` 사용
  - **역할별 토큰**: 배지 10 / 캡션·행 설명 11 / 우측 값·링크·칩·버튼 13 / 본문·토스트 14 / 행 제목 15 / 네비·화면 타이틀 17 / 섹션 타이틀 22 / 대제목 28 (상세: `docs/guide/dev/ui-publishing.md`)
  - **weight는 최대 600** — `.native.jsx`에 `fontWeight: '700'` 쓰지 말 것
  - Tailwind 단위(`px-7` 등)는 고정 픽셀 — 패딩/마진/gap에만
- **아이콘**: `@tabler/icons-react-native` + `lucide-react-native` **둘 다 사용 가능**(혼용 허용). 단 **핸드오프마다 어느 세트인지 명시**하고, **한 컴포넌트/섹션 안에서는 한 세트로 통일**할 것 (섹션 간 혼용은 OK)
- **경로 alias**: `@/` → `src/`
- `allowFontScaling={false}` 를 텍스트에 기본 적용
- 상태관리: 서버상태 TanStack Query(`src/hooks` 훅 + `src/api` fetch), 클라 상태 Zustand(`src/store`) — 디자인은 몰라도 되지만 컴포넌트를 **props/훅 주입식**으로 잡아주면 연동이 쉬움

## 3. 가장 좋은 핸드오프 포맷 (지난 편의정보 방식 = 👍)

1. **RN 변환본**(`.native.jsx`) — 컴포넌트 구조 + 스타일. 위 제약 반영해주면 거의 그대로 사용
2. **스크린샷**(full) — 최종 비주얼 기준
3. (선택) static HTML

## 4. 디자인만으로는 못 정하는 것 (연동 단계에서 판단)

- **실제 API 데이터 형태/편차** — 예: `usetime`은 스팟마다 "상시 개방"·시간범위·미사스케줄 등 제각각(구조화 대상 ~1%). 목업의 "이상적 데이터"와 실제가 다름
- **상태 기반 규칙** — 예: 편의 항목 색은 항목별 고정색이 아니라 **상태(가능=초록/없음=중립/미제공=회색)** 기준. "정보 있으면 어떤 색"인지 규칙을 명시해주면 좋음
- **로딩/에러/빈 상태** — 스켈레톤·플레이스홀더·재시도

→ 이 항목들은 "예시 데이터 1개"가 아니라 **가능한 값의 종류와 각 상태의 표현 규칙**으로 적어주세요.
