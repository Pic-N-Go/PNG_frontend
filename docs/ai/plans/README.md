# 구현 계획 문서 폴더

기능별 구현 계획을 `02-implementation-plan-template.md` 기반으로 작성해 저장합니다.

> **브랜치 기반 구조**: 파일은 브랜치명 하위 폴더에 저장합니다.
> 예: `docs/ai/plans/feature/auth-ui-publishing-parity/<feature-name>-plan.md`
> `feature/`는 예시일 뿐, `.github/CONVENTIONS.md`의 모든 브랜치 타입(`fix/`, `chore/`, `docs/`, `refactor/`, `style/`, `test/`)에 동일 패턴(`<타입>/<브랜치명>/`)이 적용됩니다.

예시 파일명:

- `feature/my-branch/login-flow-plan.md`
- `feature/my-branch/spot-detail-tabs-plan.md`

> **브랜치 없이 작업한 경우**: `main/<feature-name>-plan.md`에 저장합니다.
