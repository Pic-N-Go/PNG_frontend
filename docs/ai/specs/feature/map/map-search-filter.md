# 기능 스펙 - 지도 검색/필터 및 편의 기능 고도화

## 1) 기능 정보

- 기능명: 지도 검색/필터 UI 고도화, 필터 모달(바텀시트) 연동 및 편의 기능(줌, 내위치) 구현
- 담당자: AI 어시스턴트
- 관련 이슈: 없음
- 관련 도메인: `search`, `spot`
- 대상 플랫폼: iOS / Android

## 2) 문제와 목표

- 해결하려는 문제:
  - 현재 지도 탭의 검색바와 카테고리 필터 디자인이 홈 탭의 디자인 및 기획 디자인 가이드(목업)와 다소 상이함.
  - 홈 탭과 달리 지도 탭 검색바에는 상세 조건(거리, 시간대, 날씨 등)을 설정할 수 있는 필터 상세 모달(바텀시트) 연동이 누락됨.
  - 지도 축척을 확대/축소하는 줌 버튼(`+`, `-`) 및 내 위치로 이동하는 편의 버튼이 없어 지도 조작이 불편함.
- 사용자 가치:
  - 기획 목업과 100% 일치하면서도 홈 탭과 완벽히 호환되는 일관된 검색/필터 경험을 선사함.
  - 마커 필터링(거리, 시간대, 날씨 등)을 통해 원하는 스팟을 더 쉽고 정교하게 탐색 가능함.
  - 터치 조작 외에 버튼 기반의 지도 줌 인/아웃 및 현재 위치 매핑 기능을 제공하여 사용성을 향상함.
- 완료 기준(한 줄):
  - 상세 필터 모달(바텀시트)이 추가된 목업 준수 검색/필터 디자인을 구현하고, 카카오 맵 WebView와 연동되는 줌 컨트롤 및 내 위치 버튼 구현 완료.

## 3) 범위

- 포함(In Scope):
  - **상단 검색 바 및 뒤로가기 버튼**:
    - 뒤로가기 버튼: 원형 흰색 버튼, 그림자(`shadow-sm` 또는 `box-shadow` 토큰), ChevronLeft 아이콘 적용.
    - 검색바: 둥근 모서리(`rounded-full`), 돋보기 아이콘, 플레이스홀더 `장소, 테마, 키워드 검색`.
    - **필터 아이콘 추가**: 검색바 우측에 `IconAdjustmentsHorizontal` 버튼을 배치하여 홈 화면 검색바와 일치시킴.
    - 검색바 영역 클릭 시 `SearchResult` 화면으로 이동, 필터 아이콘 클릭 시 상세 필터 바텀시트 오픈.
  - **상세 필터 모달 (바텀시트)**:
    - 필터 버튼 터치 시 기존 `FilterBottomSheet`가 팝업되도록 구현.
    - 선택한 조건(거리, 시간대, 날씨 등)을 통해 마커 데이터를 필터링할 수 있도록 설계 (현재는 목업 연동 상태이므로 로컬 상태 변화 반영 검증).
  - **카테고리 필터**:
    - 가로 스크롤 칩 형태로 구성 (`전체`, `야경`, `바다`, `한옥`, `꽃`, `카페`, `숲`, `축제`).
    - 선택 시 배경색이 반전되는 활성화 효과 구현 (활성화: 검정 배경에 흰색 글씨, 비활성화: 연회색 배경에 회색 글씨).
    - 가로 스크롤바 숨김 처리 (`showsHorizontalScrollIndicator={false}`).
  - **우측 지도 편의 컨트롤**:
    - 줌 인(`+`) / 줌 아웃(`-`) 버튼이 세로로 연결된 카드 형태로 구성.
    - 내 위치 버튼: 줌 카드 아래 별도 카드로 배치, 조준경(Crosshair) 아이콘 적용.
    - WebView 카카오 맵 API와 `injectJavaScript`를 통해 상호작용하여 줌 레벨 조정 및 지도 중심 이동 처리.
- 제외(Out of Scope):
  - 실제 GPS 센서를 활용한 실시간 백그라운드 위치 추적 (현재 위치는 WebView 내 브라우저 navigator.geolocation 기능을 호출하는 수준으로 처리).

## 4) 사용자 시나리오

- 시나리오 A (지도 검색바 터치):
  - Given: 사용자가 지도 탭(MapScreen)을 열어 상단 오버레이를 보고 있음.
  - When: "장소, 테마, 키워드 검색"이라고 적힌 검색바 영역을 터치함.
  - Then: 검색 결과 화면(`SearchResultScreen`)으로 부드럽게 네비게이션이 작동함.
- 시나리오 B (상세 필터 모달 호출 및 적용):
  - Given: 사용자가 지도 탭에서 검색바의 필터 조절 아이콘을 탭함.
  - When: 거리 3km, 날씨 맑음 조건을 선택하고 적용 버튼을 누름.
  - Then: 필터 개수 배지가 업데이트되고, 지도상의 마커가 필터링 조건에 부합하는 스팟만 렌더링되도록 트리거됨.
- 시나리오 C (지도 줌 조작):
  - Given: 사용자가 지도를 조작하고 있음.
  - When: 우측 상단의 `+` 버튼을 터치함.
  - Then: 지도의 줌 레벨이 1단계 확대됨.
- 시나리오 D (내 위치 매핑):
  - Given: 사용자가 지도를 조작하여 다른 위치를 보고 있음.
  - When: 우측 내 위치 버튼을 터치함.
  - Then: 지도의 중심이 사용자의 현재 위치(또는 기본 기획 중심점)로 이동함.

## 5) UI/UX 요구사항

- 참조 목업 파일:
  - `src/components/ui/home/map.html` (라인 221~266)
  - `src/components/ui/home/home.html` (필터 바텀시트 UI 구조 참조)
- 화면 전환 규칙:
  - 검색바 터치 시 `SearchResultScreen`으로 Stack Navigation 이동.
  - 필터 버튼 터치 시 `FilterBottomSheet` 오픈.
- 빈 상태/에러 상태: N/A
- 로딩 상태: WebView 로딩 완료 전에는 지도 컨트롤 클릭 비활성화.

## 6) 데이터/API 요구사항

- 사용 API: 카카오 맵 JS SDK (WebView 내에서 실행)
  - 줌 인/아웃: `map.setLevel(map.getLevel() - 1)`, `map.setLevel(map.getLevel() + 1)`
  - 내 위치 이동: `map.setCenter(new kakao.maps.LatLng(lat, lng))`
- 실패 처리 방식: API 키 미설정 시 대체 에러 뷰 노출.

## 7) 상태 관리

- 서버 상태: 없음
- 클라이언트 상태(Zustand): `useTravelStore` (기존 마커 저장 상태 유지)
- 로컬 UI 상태: `filterVisible` (상세 필터 모달 오픈 여부), `activeFilterCount` (활성화된 상세 필터 수)

## 9) 수용 기준 (Acceptance Criteria)

- [ ] AC1: 상단 검색바가 홈 화면과 동일하게 필터 조절 아이콘(`IconAdjustmentsHorizontal`) 및 활성 필터 개수 배지를 노출하며, 뒤로가기 버튼 디자인이 목업과 일치함.
- [ ] AC2: 카테고리 스크롤 필터가 목업의 폰트 및 둥근 칩 디자인을 유지하며 가로 스크롤로 정상 작동함.
- [ ] AC3: 필터 아이콘 터치 시 `FilterBottomSheet`가 정상 마운트되고, 적용 결과에 따라 필터 배지 수치가 갱신됨.
- [ ] AC4: 우측 줌 인/아웃 버튼 및 내 위치 버튼 레이아웃이 깔끔하게 띄워지며 터치 시 WebView 내 지도가 연동되어 실시간으로 확대/축소/중심 이동이 이뤄짐.

## 10) 테스트 시나리오

- 정상 케이스:
  - 줌 인/아웃 버튼 터치 시 지도가 확대/축소됨.
  - 필터 조절 버튼 터치 시 바텀시트가 열리고 조건 선택 후 적용 시 정상 작동함.
- 경계 케이스:
  - 카카오 맵 최대/최소 줌 레벨 도달 시 줌 조작 예외 발생 방지.

## 11) 오픈 이슈 / 결정 필요

- 상세 필터에 따른 마커 필터링 로직: 현재 지도 마커 데이터가 Mock 리스트이므로 상세 필터 적용 시 마커 배열을 filter 처리하여 WebView 내부에 재주입하는 기능 구현 필요.
