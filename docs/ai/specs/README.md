# 기능 스펙 문서 폴더

기능별 스펙을 `01-feature-spec-template.md` 기반으로 작성해 저장합니다.

> **브랜치 기반 구조**: 파일은 브랜치명 하위 폴더에 저장합니다.
> 예: `docs/ai/specs/feature/auth-ui-publishing-parity/<feature-name>.md`
> `feature/`는 예시일 뿐, `.github/CONVENTIONS.md`의 모든 브랜치 타입(`fix/`, `chore/`, `docs/`, `refactor/`, `style/`, `test/`)에 동일 패턴(`<타입>/<브랜치명>/`)이 적용됩니다.

예시 파일명:

- `feature/my-branch/login-flow.md`
- `feature/my-branch/spot-detail-tabs.md`

> **브랜치 없이 작업한 경우**: `main/<feature-name>.md`에 저장합니다.
